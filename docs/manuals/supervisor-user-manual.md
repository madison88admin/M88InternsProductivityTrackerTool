## 1. System Overview

The M88 Interns Productivity Tracker is a workflow platform for attendance, tasks, narratives, approvals, and allowance processing.

Supervisor function in the system:
- review and decide pending records
- monitor team attendance integrity
- maintain task quality and deadlines
- escalate unresolved issues appropriately

## 2. Getting Started

### 2.1 Logging In

1. Open system URL.
2. Enter supervisor email and password.
3. Tick Terms and Conditions checkbox.
4. Click Sign In.

Expected result:
- supervisor dashboard loads
- supervisor sidebar appears

### 2.2 Setting Your Password (New Users)

1. Open invite link.
2. Enter and confirm new password.
3. Complete setup.
4. Login using new password.

### 2.3 Resetting Your Password

1. Click Forgot Password.
2. Enter email.
3. Open reset link from mailbox.
4. Set and confirm new password.
5. Login again.

### 2.4 Logging Out

1. Click Sign Out.
2. Wait for login page redirect.

## 4. Supervisor Role Guide

### 4.1 Role Overview

Supervisor role objectives:
- ensure submission quality
- reduce approval turnaround time
- document clear reasons for rejection
- escalate risk cases quickly

Daily priority sequence:
1. Approvals queue
2. Team Attendance anomalies
3. Team Narratives quality
4. Task status health

### 4.2 Dashboard

Purpose:
- immediate summary of pending volume and team status

Use process:
1. check pending counts
2. identify urgent indicators
3. proceed to Approvals module

[SCREENSHOT-SUP-004: Supervisor dashboard KPI cards]

### 4.3 Approvals

Purpose:
- main decision center for attendance, narrative, task, and correction records

Standard workflow:
1. Open Approvals.
2. Filter by status/type/date.
3. Open first pending record.
4. Validate details and consistency.
5. Approve or Reject.
6. Add clear comments.
7. Continue until queue is reduced.

Bulk approve workflow:
1. filter to safe homogeneous records
2. confirm no restricted correction type is mixed
3. click Approve All
4. confirm result

Decision checklist before approval:
- data is complete
- date/session context is valid
- no obvious conflict with attendance/task history
- submission follows policy

Rejection best practice:
- state exact issue
- state required correction
- state next step for resubmission

### 4.4 Task Management

Purpose:
- assign and govern intern tasks in supervisor scope

Create task:
1. click Create Task
2. enter title and objective
3. add clear description and expected output
4. assign intern
5. set due date and estimated hours
6. save

Edit task:
1. open task
2. click Edit
3. update permitted fields
4. save

Archive task:
1. filter completed tasks
2. choose Archive
3. confirm

Operational standards:
- write measurable and clear task instructions
- monitor overdue tasks daily
- avoid unnecessary rework through clear acceptance criteria

### 4.5 Team Attendance

Purpose:
- monitor attendance quality and resolve missing entries under policy

Workflow:
1. open Team Attendance
2. choose week/date filters
3. review rows for missing punches, late flags, and IP issues
4. process allowed actions

Fill end-of-day workflow (if allowed):
1. open eligible past record
2. confirm prerequisite punches exist
3. fill missing value
4. save and verify

Escalation workflow:
1. open problematic record
2. click Escalate
3. enter reason
4. submit

After escalation, further edits may be restricted.

### 4.6 Team Narratives

Purpose:
- evaluate narrative quality and approve/reject submissions

Workflow:
1. open Team Narratives
2. apply filters (intern, status, session, date)
3. read full narrative
4. approve or reject with guidance

Approval criteria:
- clear and complete content
- aligned with task and attendance context
- actionable output description

Reject when:
- vague or incomplete
- inconsistent details
- missing required context

## 5. Frequently Asked Questions and Troubleshooting

### FAQ

Q: Why is bulk approve disabled?  
A: Current filter may include restricted item type or no eligible pending records.

Q: Why can I not edit a team attendance row?  
A: Record may be escalated or locked by workflow rules.

Q: Why is intern not visible in my list?  
A: Assignment/scope rules may not include that intern.

Q: How detailed should rejection comments be?  
A: Enough to tell intern exactly what to correct and how to resubmit.

### Troubleshooting Table

| Issue | Possible Cause | Resolution |
|---|---|---|
| Cannot approve record | already processed | refresh and confirm latest status |
| Bulk actions unavailable | restricted mixed queue | narrow filters and process restricted types individually |
| Team record missing | scope mismatch | confirm assignment with admin |
| Unable to update attendance | escalated/locked state | escalate to admin for final handling |
| Empty narrative list | over-filtered view | clear filters and reload |
