# M88 Interns Productivity Tracker
## Screenshot Capture Guide for User Manuals

Version: 1.0  
Date: April 6, 2026

---

## Purpose

This guide standardizes screenshots for the three user manuals:
- Intern User Manual
- Supervisor User Manual
- Administrator User Manual

---

## Capture Standards

1. Use desktop browser view (recommended 1366x768 or 1920x1080).
2. Capture full module area including page title and key actions.
3. Keep personal data masked when possible.
4. Use consistent zoom level (100 percent) unless showing zoom feature.
5. Save images as PNG for clarity.
6. Use light mode and default system theme for consistency.
7. Include visible success/error messages when the step references them.

---

## File Naming Standard

Use this format exactly:
- intern: INT-001, INT-002, ...
- supervisor: SUP-001, SUP-002, ...
- admin: ADM-001, ADM-002, ...

Recommended full file format:
- INT-001-login-page.png
- SUP-004-approvals-queue.png
- ADM-020-system-settings.png

## Recommended Capture Sequence

1. Capture all shared pages first (login, notifications, profile).
2. Capture intern-only workflows end-to-end.
3. Capture supervisor approvals and team pages.
4. Capture admin operations and system pages last.
5. Review names and order against manual placeholders before final insertion.

---

## Word Document Assembly Tips

1. Paste screenshots directly below their matching [SCREENSHOT-...] marker.
2. Add a caption under each image:
   Figure X. [Short title]
3. Keep image width consistent per manual.
4. Use page breaks before each major module section.
5. After insertion, update table of contents in Word.

---

## Quality Check Before Final Submission

- all screenshot markers in manuals have matching images
- no confidential or personal data exposed
- all labels, statuses, and buttons are readable
- image order matches procedure steps
- manual version/date matches release date
